# Web Designer SEO Framework (Inspired by Matt Diggity)

This guide serves as a protocol for the agent to ensure every website build is optimized for search engine dominance from the ground up.

## 1. Site Architecture (The Silo Strategy)
Architecture is the foundation of topical authority. We must avoid "Yolo Silos" (random linking) and instead use structured hierarchies.

*   **Reverse Silo (Topical Authority):** Link between related blog posts and informational pages to build a "web" of relevance around a specific topic.
*   **Priority Silo (The Money Move):** Direct the strongest internal links toward "Money Pages" (Service pages, Product pages, CTAs). Ensure "link juice" flows where conversions happen.
*   **No Dead Ends:** Every single page must contain at least one internal link to another relevant page on the site.
*   **Contextual Linking:** Links should be placed within the body content, not just in navigations or footers, to signal maximum relevance to Google.

## 2. On-Page SEO Fundamentals
*   **Front-Loaded Title Tags:** Place the primary target keyword at the very beginning of the `<title>` tag.
*   **The "First 100 Words" Rule:** The primary keyword must appear within the first 100 words of the page content to establish immediate topical relevance.
*   **Short, Keyword-Rich URLs:** Keep URLs concise. Aim for a maximum of 3 words (e.g., `/anesthesia-record-app` instead of `/the-best-anesthesia-record-app-for-dentists`).
*   **Heading Strategy:** Use H1 for the main keyword, and H2/H3 for supporting semantic entities. Avoid over-optimizing with the same exact phrase; use variations.

## 3. NLP & Entity Alignment
*   **Natural Language Processing (NLP):** Align content with the entities and phrases Google expect to see in a high-ranking article for that niche. (e.g., For "Anesthesia," include "Vitals," "Sedation," "Patient Safety," "Compliance").
*   **Entity Density:** Ensure the "Secondary Keywords" identified in competitor analysis are naturally distributed throughout the body text.

## 4. Performance & Technical Standards
*   **Zero-Lag Rendering:** Prioritize Static Site Generation (SSG). A faster site is a higher-ranking site.
*   **Mobile-First Design:** Ensure every element is optimized for touch and small screens (Astro's responsive components are mandatory).
*   **Clean Code:** Minimize unused CSS and JS. Google rewards "lean" pages that are easy to crawl.

## 5. Ongoing Optimization Protocol
*   **SEO is Not "One and Done":** Content should be reviewed and re-optimized based on ranking performance.
*   **Post-Launch Validation:** Run the technical SEO validation script (from the TSOP) after every major update to ensure no metadata or alt tags were dropped.
